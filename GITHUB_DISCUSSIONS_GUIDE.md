## Discussion Guidelines

### Before Posting

1. **Search First**: Check if your topic has been discussed
2. **Choose Right Category**: Select the most appropriate category
3. **Clear Title**: Descriptive, specific title
4. **Provide Context**: Background information and details
5. **Be Respectful**: Follow code of conduct

### Writing Good Discussions

**Good Title**:
- ‚úÖ "Proposal: Add multi-language support to AI Legal Hotline"
- ‚ùå "Question about hotline"

**Good Description**:
- Clear problem statement
- Specific details and context
- Relevant links and references
- What you've tried (for questions)
- Expected vs. actual outcomes

**Good Follow-up**:
- Respond to questions
- Provide updates
- Mark solutions
- Thank contributors

### Engaging with Discussions

**Do**:
- ‚úÖ Ask clarifying questions
- ‚úÖ Share relevant experience
- ‚úÖ Provide constructive feedback
- ‚úÖ Link to related discussions
- ‚úÖ Upvote helpful content

**Don't**:
- ‚ùå Go off-topic
- ‚ùå Make personal attacks
- ‚ùå Spam or self-promote
- ‚ùå Share private information
- ‚ùå Duplicate discussions

## Special Discussion Types

### RFC (Request for Comments)

**Purpose**: Formal proposals for major changes  
**Process**:
1. Draft RFC document
2. Post in Ideas category with [RFC] tag
3. 14-day comment period
4. Address feedback
5. Core Team decision
6. Implementation or rejection

**Template**:
```markdown
# [RFC] Title

## Summary
One-paragraph explanation

## Motivation
Why are we doing this?

## Detailed Design
How will this work?

## Drawbacks
What are the downsides?

## Alternatives
What other approaches were considered?

## Unresolved Questions
What needs to be figured out?
```

## Best Practices

### For Askers

1. **Be Specific**: Provide enough detail for others to help
2. **Show Your Work**: What have you tried?
3. **Accept Answers**: Mark helpful responses
4. **Follow Up**: Share outcomes and learnings
5. **Pay It Forward**: Help others when you can

### For Responders

1. **Be Helpful**: Provide actionable guidance
2. **Be Patient**: Not everyone has same expertise
3. **Be Respectful**: Assume good intentions
4. **Link Resources**: Share relevant documentation
5. **Ask Questions**: Clarify before answering

### For Everyone

1. **Stay On Topic**: Keep discussions focused
2. **Be Constructive**: Critique ideas, not people
3. **Share Knowledge**: Document solutions
4. **Build Community**: Welcome newcomers
5. **Follow Up**: Close loops and share outcomes

## Discussion Lifecycle

### Active Discussions
- Regular engagement
- Questions being answered
- Ideas being refined
- Decisions being made

### Resolved Discussions
- Question answered (mark solution)
- Idea implemented or rejected
- Decision made and documented
- Lock discussion with summary

### Stale Discussions
- No activity for 30+ days
- Automatic "stale" label
- Reminder to close or update
- Auto-close after 60 days (unless pinned)

## Examples of Great Discussions

### Great Question
**Title**: "How to integrate Sentinel detection with existing law enforcement systems?"

**Content**:
```markdown
I'm working on integrating the Sentinel detection system with our 
department's existing case management system. 

**Current Setup**:
- Case management: Vendor XYZ v3.2
- Database: SQL Server 2019
- Authentication: Active Directory

**What I've Tried**:
- Reviewed API documentation
- Tested REST endpoints
- Encountered authentication issues

**Specific Question**:
How should I handle authentication between Sentinel and our AD system?

**Environment**:
- OS: Windows Server 2022
- Network: Internal only
- Security: High compliance requirements
```

### Great Idea
**Title**: "[Idea] Add real-time translation to AI Legal Hotline"

**Content**:
```markdown
## Problem Statement
Many victims don't speak English as their primary language, limiting 
access to the AI Legal Hotline.

## Proposed Solution
Integrate real-time translation API to support 20+ languages.

## Benefits
- Increased accessibility
- Better victim support
- Broader reach
- Equity improvement

## Implementation Considerations
- Translation API costs
- Accuracy requirements
- Privacy concerns
- Technical integration

## Alternatives Considered
- Human translators (expensive, limited availability)
- Pre-recorded messages (inflexible)
- Text-only translation (excludes voice)

## Next Steps
Looking for feedback on:
1. Which translation API to use?
2. How to handle legal terminology?
3. Privacy and security considerations?
```
